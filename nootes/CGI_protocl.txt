
Common Gateway Interface

os cgi e usado quando a requesisao http pede execusao de um script que retorna a mensagem para o server enviar.

O cgi classico recebe or as vairavei ja tratadas or o um pedaco do request e tira as variaveis 
e em segida executa o uma instacia do programa a executar (.py, ./ola, etc)enviando as env para ele. 

A umas variaveis base que ele usa sao obirgatorias.

Se as vairavei nao vierem bem retornas error(404) page ou 500 (ver as circonstacias).

Variável	Descrição
REQUEST_METHOD	Método HTTP da requisição (GET, POST, HEAD, etc.)
QUERY_STRING	Dados enviados na URL (quando método é GET)
CONTENT_TYPE	Tipo de dados no corpo da requisição (ex.: application/x-www-form-urlencoded, application/json)
CONTENT_LENGTH	Tamanho do corpo da requisição (importante em POST)
HTTP_COOKIE	Cookies enviados pelo navegador, no formato chave=valor
HTTP_USER_AGENT	Identificação do navegador ou cliente que fez a requisição
HTTP_REFERER	URL da página que originou a requisição (se existir)
REMOTE_ADDR	Endereço IP do cliente
REMOTE_HOST	Nome DNS do cliente (se o servidor fizer resolução reversa)
SCRIPT_NAME	Nome do script CGI chamado
SCRIPT_FILENAME	Caminho completo do script no servidor
PATH_INFO	Informação de caminho extra passada após o nome do script
SERVER_NAME	Nome do servidor (hostname ou IP)
SERVER_PORT	Porta do servidor usada na requisição
SERVER_PROTOCOL	Protocolo usado, ex.: HTTP/1.1
SERVER_SOFTWARE	Nome e versão do servidor web (ex.: Apache/2.4.41)


Variável obirgatorias

REQUEST_METHOD → método HTTP usado (ex.: GET, POST)
SCRIPT_NAME → caminho “lógico” do script dentro do site
SERVER_NAME → nome do host ou IP do servidor
SERVER_PORT → porta onde o servidor recebeu a requisição
SERVER_PROTOCOL → protocolo e versão usados (ex.: HTTP/1.1)
